challenges i faced whike making my project 
1. Uploaded files weren't ready to use right after uploading after upload the file was saved but the system never actually read it or processed it so when the chat tried to use it it had nothing to work with. I fixed it by making the upload automatically go through the whole pipeline, read the text split it embed it before calling it done.
2. Steps were running in the wrong order different parts of the pipeline weren't connected properly so sometimes the system would try to search a document before it was even processed. I linked each step to the next one so they always run in the right order.
3. Retrieval was failing but nobody could tell when the search part broke or returned nothing useful the LLM would still try to answer and just make something up that sounded real. I added a check that catches this and shows the error instead of letting the chat keep going with a fake answer.
4. Uploading the same file twice created duplicates if someone uploaded the same resume twice the system would save it again, process it again and add all the same content to the index again. I made it so the same content only gets processed once no matter how many times someone uploads it.
5. The same chunk was getting added to the index more than once even if the file had a different name the same text could still get embedded again and end up duplicated in the index. I keep a running list of everything already processed and skip anything that is already on that list.
6. Figuring out how to split documents into chunks i had to decide how to break documents into pieces before embedding them and getting it wrong directly hurt search quality. I went with splitting by paragraphs with a little overlap between them which worked better for resumes since they are already organized into natural sections.
7. Finding an embedding model that worked locally i needed something to turn text into vectors but didn't want to pay for an API or depend on the internet. I used nomic embed text through Ollama which runs fully on my machine for free and was easy to plug in.
8. Finding a local LLM for generating answers same problem on the answer generation side, I wanted everything running locally without cloud dependencies. I used llama3 8b Ollama which kept the whole pipeline on my machine and made it easy to test and debug.
